# Действия владельца после деплоя

## 1. Первоначальная настройка

### 1.1. Получение Telegram API credentials
1. Перейдите на https://my.telegram.org/apps
2. Создайте новое приложение
3. Скопируйте `API_ID` и `API_HASH` в `.env` файл

### 1.2. Создание бота
1. Напишите @BotFather в Telegram
2. Создайте нового бота командой `/newbot`
3. Скопируйте токен бота в `BOT_TOKEN` в `.env`

### 1.3. Настройка базы данных
1. Убедитесь, что PostgreSQL запущен
2. Обновите `DATABASE_URL` в `.env` с правильными credentials

### 1.4. Генерация master ключа
```bash
# Linux/Mac
openssl rand -hex 32

# Windows (PowerShell)
[Convert]::ToHexString((New-Object byte[] 32 | %{ (New-Object System.Security.Cryptography.RNGCryptoServiceProvider).GetBytes($_) }))
```
Скопируйте результат в `MASTER_KEY` в `.env`

## 2. Запуск системы

### 2.1. Запуск всех сервисов
```bash
make up
```

### 2.2. Применение миграций
```bash
make migrate
```

### 2.3. Инициализация данных
```bash
make seed
```

## 3. Настройка через бота

1. Начните чат с вашим ботом
2. Отправьте `/start` для регистрации администратора
3. Подключите master-аккаунт:
   - Отправьте номер телефона
   - Введите код подтверждения
   - Введите пароль 2FA (если установлен)
4. Выберите каналы для отслеживания из списка подписок master
5. Укажите целевой канал для публикации дайджестов
6. Проведите тестовую публикацию

## 4. Проверка работы

1. Убедитесь, что все контейнеры запущены:
   ```bash
   docker compose ps
   ```
2. Проверьте логи:
   ```bash
   make logs
   ```
3. Отправьте тестовый пост в один из отслеживаемых каналов
4. Проверьте, что пост появился в очереди модерации в боте
